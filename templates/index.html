Open Name System

ICANN DNS + blockchain handles + blockchain domains

OVERVIEW

The Open Name System (ONS) is a protocol that extends DNS by supporting the registration and resolution of:

  1. user handles on the blockchain (DNS for people)
  2. domains on the blockchain (truly decentralized DNS)

OPENDIG

Opendig is a command line tool that extends the functionality of dig to support the resolution of user handles registered on the blockchain.

Domains:

$ opendig -d angel.co

angel.co. 59 IN A 50.18.171.72
angel.co. 21599 IN NS ns-1437.awsdns-51.org.
angel.co. 21599 IN NS ns-1845.awsdns-38.co.uk.
angel.co. 21599 IN NS ns-193.awsdns-24.com.
angel.co. 21599 IN NS ns-634.awsdns-15.net.
...

User Data:

$ opendig -u naval

{
    "name": {
        "formatted": "Naval Ravikant"
    },
    "website": "https://angel.co/naval",
    "bitcoin": {
        "address": "1919UrhYyhs471ps8CFcJ3DRpWSda8qtSk"
    },
    ...
    }
}

Bitcoin User Data:

$ opendig bitcoin -u naval

{
        "address": "1919UrhYyhs471ps8CFcJ3DRpWSda8qtSk"
}

USER SCHEMA

User calls to opendig will return user data in a standard schema - today most of the data is in what is known as userschema v0.2, but a new standard v0.3 is in the RFC phase. After sufficient people have reviewed, commented on, and revised the v0.3 schema, a rollout will begin.
